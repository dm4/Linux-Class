什麼是 Shell

  • Shell 是一支程式

  • 指令直譯器

  • 提供方便的文字介面讓使用者與作業系統溝通

  • $ command [Options] argument1 argument2 ...

為什麼要用 Shell

  • 文字介面

    • 簡化和電腦的互動

  • 遠端操作

    • 只需要傳送文字資料，速度快

  • 可程式化

常見的 Shell

  • Bourne shell
    • sh
    • 由 Stephen Bourne 開發於 1977 年 (AT&T Bell lab) 是Unix系統的預設的 Shell
  • C shell
    • csh
    • 1970年 Bill Joy 開發於柏克萊加州分校在 BSD 系統上，類似 C 語言

常見的 Shell

  • Tenex C shell
    • tcsh
    • C shell 的加強版本
  • Bourne Again shell
    • bash
    • Brian Fox 開發於 1987 年，支援整數運算、I/O 重定向等更多功能
    • Ubuntu/Debian 預設
  • Other shells
    • zsh, fish, ...

Shell 的文字介面

  • 一打開 Terminal 看到的就是 shell 的介面

    user@ubuntu:~$

  • 不同的 shell 可能會有不同的介面

更換 Shell

  • 查看目前 shell

    $ echo $SHELL

  • 查看系統可用的 shell

    $ cat /etc/shells

  • 更換 shell

    $ chsh

協助資訊

  • man

    CP(1)
NAME
       cp - copy files and directories

  • CP(1) 中的 1 代表指令在 manual 的分類

協助資訊

  • 1 使用者在 shell 環境中可以操作的指令或可執行檔
  • 2 系統核心可呼叫的函數與工具等
  • 3 一些常用的函數 (function) 與函式庫 (library) ，大部分為 C 的函式庫 (libc)
  • 4 裝置檔案的說明，通常在 /dev 下的檔案
  • 5 設定檔或者是某些檔案的格式
  • 6 遊戲 (games)
  • 7 慣例與協定等，例如 Linux 檔案系統、網路協定、ASCII code 等等的說明
  • 8 系統管理員可用的管理指令
  • 9 跟 kernel 有關的文件

為什麼要用套件管理

    • 不用上網找安裝檔

  • 套件相依性

    • 程式A 需要用到 程式B
    • 程式B 需要用到 程式C
    • ...

不同系統的套件管理

  • rpm, yum
    • RedHat, CentOS, ...
  • dpkg, apt
    • Debian, Ubuntu
  • ports
    • FreeBSD
  • ...

Ubuntu 的套件管理

  • dpkg
    • 低階套件工具
    • 單純安裝／移除套件
  • apt
    • 中階套件工具
  • aptitude
    • 高階套件工具

Ubuntu 的套件管理

apt

  • 更新套件庫

    $ sudo apt-get update

  • 安裝套件

    $ sudo apt-get install [套件名稱]

  • 移除套件

    $ sudo apt-get remove [套件名稱]

apt

  • 更新套件

    $ sudo apt-get upgrade

  • 搜尋套件

    $ apt-cache search [套件名稱]

aptitude

  • 好處

    • 聰明的 install/remove
    • 較好的 dependency 判斷

  • 安裝 aptitude

    $ sudo apt-get install aptitude

aptitude

  • 安裝套件

    $ sudo aptitude install [套件名稱]

  • 移除套件

    $ sudo aptitude remove [套件名稱]
$ sudo aptitude purge [套件名稱]

aptitude

  • 更新套件

    $ sudo aptitude upgrade
$ sudo aptitude safe-upgrade  ->  用這個比較好！

  • 搜尋套件

    $ sudo aptitude search [套件名稱]

使用 bash

alias

  • 為指令取別名
  • 把長指令「縮寫」
  • alias la="ls -al"
  • alias rm="rm -i"
  • 不加參數會列出縮寫過的指令
    • alias

bash 的環境變數

  • 可用 export 來檢視目前的環境變數
  • 可用 export VAR="DATA" 來自訂環境變數
  • 環境變數和 alias 的縮寫 不一樣

bash 的設定檔

  • /etc/profile
    • 為系統公用的使用者設定檔，在使用者登入時被讀取
  • /etc/bash.bashrc
    • 為系統公用的 bash 設定檔，在 bash 被執行時被讀取
  • ~/.profile
    • 使用者私有的設定檔
  • ~/.bashrc
    • 使用者私有的 bash 設定檔

bash 的指令歷史記錄

  • bash, csh, tcsh 均支援指令記錄
  • historoy 可看指令歷史記錄
  • ![編號] 可重做指令

標準輸入／輸出 (Standard I/O)

  • 標準輸入／輸出／錯誤輪出
  • stdin／stdout／stderr

  • 標準輸入(stdin)

    • 指令的資料來源，通常是終端機的鍵盤輸入，以檔案描述詞 0 表示

  • 標準輸出(stdout)

    • 指令的執行結果，通常會輸出至終端機的螢幕上，以檔案描述詞 1 表示

  • 標準錯誤輸出(stderr)

    • 指令執行時產生的錯誤訊息，通常會輸出至終端機的螢幕上，為了有別於正常的標準輸出，以檔案描述詞 2 表示

I/O Redirect 和 Pipeline

  • 把結果輸出至檔案

    • $ 指令 > 檔案
    • $ 指令 >> 檔案 附加於檔案後

  • 把檔案當成輸入讀進來

    • $ 指令 < 檔案

  • 把一個指令的結果當成另一個指令的輸入

    • $ 指令1 | 指令2
